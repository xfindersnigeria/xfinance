  Practical test plan

  Fresh bank account, no prior data — do these in order:

  Test 1 — Bank account, no opening balance

  1. Create a new bank account (e.g., "Test Bank GTB")
  2. Open it — balance should show ₦0.00
  3. Create an invoice for a customer for ₦50,000 → post it
    - Expected: A/R balance = 50,000 (positive) ✓
    - Bank balance still = 0 (no cash received yet)
  4. Record customer payment of ₦50,000 against that invoice
    - Expected: Bank balance = ₦50,000 ✓, A/R balance = 0
  5. Create a vendor expense (no bill) of ₦10,000 paid from that bank account
    - Expected: Bank balance = ₦40,000 ✓
  6. View account transactions on the bank account — each row should show:
    - Payment received: debitAmount = 50,000, creditAmount = 0, runningBalance = 50,000 ✓
    - Expense: debitAmount = 0, creditAmount = 10,000, runningBalance = 40,000 ✓

  ---
  Test 2 — Bank account, with opening balance

  1. Create another fresh bank account (e.g., "Test Bank Zenith")
  2. Go to Opening Balance, set this account with ₦100,000 debit
    - Expected: Bank balance = ₦100,000 ✓
    - Opening Balance Equity account (3140-01) = ₦100,000 (positive, equity credit-normal)
  3. Record a bill of ₦20,000, pay it from this bank account
    - Expected: Bank balance = ₦80,000 ✓
    - A/P goes from 20,000 → 0 after payment ✓
  4. Create a cash receipt (immediate sale, no invoice) of ₦15,000 into this bank
    - Expected: Bank balance = ₦95,000 ✓
  5. View account transactions — running balance column should step from 100,000 → 80,000 → 95,000 ✓

  ---
  What to watch for

  - Bank balance in the card header: always positive, increases on receipts/payments-received, decreases on expenses/payments-made
  - If bank ever goes negative (overdraft) — currently no color warning on frontend, it will just show negative number — that's a future UI improvement
  - The AccountStats "Total Debits" and "Total Credits" cards may show incorrect values for Revenue/Liability accounts (the AccountDetails.tsx issue), but not for bank
